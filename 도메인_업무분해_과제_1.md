# 온보딩 모듈 최종 과제 — 내 도메인 업무를 네 조각으로

**도메인:** 규칙 기반·역발상 투자 정보 서비스
**과정 전체에서 다룰 개인 문제 초안**

---

## 1단계 — 문제 한 가지 고르기

**고른 문제:** 매일 아침 시장 국면을 확인하고 TQQQ / 커버드콜 매매 신호를 판단하는 일

"하고 싶은 멋진 일"이 아니라 "매일 반복하는 일"이라서 골랐다. 나는 20년 넘게 같은 판단을 손으로 반복해 왔다 — 공포·탐욕 지수(F&G)를 확인하고, 정해 둔 임계값과 대조해, 공격 자산과 수비 자산을 다르게 다룬다. 판단 규칙 자체는 이미 내 머릿속에 확립돼 있다. 문제는 이 규칙이 **매일 아침 사람의 손과 눈을 거쳐야만 실행된다**는 점이다. 여행을 가도, 아파도, 감정이 흔들려도 규칙은 그대로여야 하는데 실행자는 흔들린다. 이 반복적이고 규칙이 명확한 작업이야말로 AI 에이전트에게 넘기기 가장 좋은 첫 후보다. 넉 달 뒤 해커톤의 최종 목표(증권사 API를 통한 자동 실행)도 결국 이 문제의 확장이다.

### 운용 구조 — 두 자산은 역할이 다르다 (하루 종일 시뮬레이션으로 검증·재보정 완료)

핵심은 커버드콜 ETF와 TQQQ를 **똑같이 다루지 않는다**는 점이다. F&G 지수는 이 둘을 어떻게 다룰지 결정하는 스위치다.

- **커버드콜 ETF = 수비 (계속 모아간다):** F&G와 무관하게 **팔지 않고 꾸준히 매집만** 한다. "2세대 커버드콜"은 나스닥100의 약 90%를 그대로 추종하면서 연 15% 안팎을 매달 배당으로 지급한다 — 옛날식 "상승 제한된 안전자산"이 아니라 실질적으로 **"배당 나오는 성장자산"**이라는 게 실제 계좌 데이터로 확인됐다.
- **TQQQ = 공격 (사이클로 매매한다):** F&G 사이클에 따라 **사고판다**. F&G≤25/20/15에서 실탄의 25%/50%/100% 분할 매수, F&G≥75/80에서 보유분의 50%/전량 분할 매도. 평소엔 대기(예수금·외화RP).
- **커버드콜 추가매수:** F&G가 극단(공포도 탐욕도 아닌) 35~65 구간일 때, 노는 실탄의 20%를 커버드콜에 추가 투입한다. 이게 하루 종일 테스트한 것 중 결과를 가장 크게 개선시킨 규칙이었다 — 실탄이 F&G≤25만 기다리며 저금리로 노는 것보다, 평시에도 계속 재투자되는 게 훨씬 낫다.

정리하면, **수비는 쌓기만 하고 공격만 사이클로 돌리되, 노는 실탄은 평시에도 수비로 흘러 들어간다.**

## 2단계 — 네 동작 워크북

| 동작 | 내 도메인 (매일 아침 매매 신호 판단) |
|---|---|
| **분해** (5~7개 작은 단위로) | ① F&G 데이터 수집 → ② 포지션 확인(TQQQ·예수금·커버드콜 적립분) → ③ 규칙 대조(TQQQ 매수 F&G≤25/20/15, 매도 F&G≥75/80, 커버드콜 매수 F&G 35~65) → ④ 신호 판정 → ⑤ 티어별 분할 수량 계산(25%/50%/100%, 50%/전량) → ⑥ 기록·알림 |
| **패턴 인식** (반복 2~3개) | · **공격 매수:** F&G≤25→25%, ≤20→50%, ≤15→100% 분할 매수<br>· **공격 매도:** F&G≥75→보유분 50%, ≥80→잔량 전량<br>· **수비 적립:** F&G 35~65(평시) → 실탄의 20% 커버드콜 추가매수 + 매달 배당 재순환 |
| **추상화** (본질 한 줄) | "수비(커버드콜, 나스닥 90%추종+배당15%)는 평시에도 계속 재투자로 몸집을 키우고, 공격(TQQQ)은 F&G 극단에서만 사이클로 사고판다" |
| **알고리즘** (단계별 순서) | 데이터 수집 → 커버드콜 적립 판정 → TQQQ 매매 판정 → 수량 계산 → 기록·알림 출력 |

> **처음엔 가장 어려운 칸이었던 것:** ⑤ 분할 실행 수량 계산. "F&G≤25면 매수"까지는 규칙이지만, 얼마를 몇 번에 나눠 실행할지는 처음엔 감이었다. → **하루 종일 시뮬레이션(30년 몬테카를로·실제 16년 역사·블록부트스트랩)으로 25%/50%/100% 3단계 티어, 매도 50%/전량 2단계 티어로 숫자를 확정했다.** 여전히 남은 어려운 칸은 **커버드콜의 정확한 수익 구조(베타·프리미엄·행사가 캡)와, "몇 %의 미래 성장률을 기본 가정으로 삼을 것인가"**다 — 최근 15년 실측(연 19.7%)은 너무 낙관적이고, 30년 전체 실측(연 14.4%)이 더 현실적이라는 결론까지는 냈지만, 이 판단 자체가 여전히 사람의 직관 영역이다.

## 3단계 — AI 전달 형식 (다음 모듈 예고편)

```
[도메인 맥락]
나는 규칙 기반·역발상 투자 정보 서비스를 운영하는 20년 차 투자자이고,
매일 아침 공포·탐욕 지수(F&G)를 확인해 두 자산의 신호를 판단하는 작업을
자동화하고 싶습니다. 핵심 철학은 감정을 배제한 기계적 규칙 실행입니다.
두 자산은 역할이 다릅니다.
 - 수비(커버드콜, 나스닥100 90%추종+배당15%): F&G 35~65 평시에도 실탄의 20%를 계속 추가매수.
 - 공격(TQQQ): F&G≤25/20/15에서 실탄의 25%/50%/100% 분할매수,
   F&G≥75/80에서 보유분의 50%/전량 분할매도.
F&G는 "지금 공격에 실탄을 넣을 때인가, 회수할 때인가"를 판정하는 스위치입니다.

[네 동작 분석]
1. 분해: F&G 수집 → 포지션 확인 → 커버드콜 매수 판정 → TQQQ 매매 판정 → 수량 계산 → 기록·알림
2. 패턴: (a) F&G≤25/20/15 → TQQQ 25%/50%/100% 분할매수
         (b) F&G≥75/80 → TQQQ 50%/전량 분할매도
         (c) F&G 35~65(평시) → 커버드콜 실탄 20% 추가매수
3. 추상화: "수비는 평시에도 계속 재투자로 몸집을 키우고, 공격은 F&G 극단에서만 사이클로 사고판다"
4. 알고리즘: 데이터 입력 → 커버드콜 적립 판정 → TQQQ 매매 판정
            → 분할 수량 산출 → 근거와 함께 리포트 출력

[출력 요구]
매일 아침 한 장짜리 신호 리포트 형식으로, 한국어로.
(오늘의 F&G 값 / 판정된 신호 / 판정 근거 / 권장 실행 수량 포함)

[검증 기준]
20년 경력의 본인이 "내가 손으로 판단한 결과와 100% 일치한다"고 평가하고,
30년 몬테카를로·실제역사 백테스트 결과(CAGR 14~15%대, MDD -45~50%대)와
정합적인 결과를 낼 것.
```

가장 중요한 칸은 맨 위 **[도메인 맥락]**이다. "공격과 수비의 두 축", "감정 배제" 같은 나만의 운용 철학을 한 단락으로 적어 주지 않으면, AI는 일반적인 매매 봇 답안을 내놓는다. 내 도메인의 진짜 깊이를 알려 줄수록 AI는 그 안에서 빠르게 일한다.

## 자기 점검 — 아직 또렷하지 않은 칸

이 형식에서 내가 가장 흐릿하게 적은 칸은 **분할 수량 계산(알고리즘 5단계)**과 **검증 기준의 정량화**다.

- **분할 수량:** "F&G ≤25면 매수"까지는 규칙이지만, 몇 %씩 몇 회에 나눌지는 아직 명문화된 규칙이 아니라 경험적 감이다. → 넉 달 과정에서 이 감을 *숫자 규칙*으로 옮기는 것이 핵심 과제.
- **검증 기준:** "본인 판단과 일치"는 주관적이다. → 과거 F&G 데이터로 백테스트해 "지난 N개 신호 중 내 실제 매매와 몇 % 일치했는가"로 바꾸면 검증이 또렷해진다.

빈 칸이 아니라 "덜 또렷한 칸"이 바로 내일 다듬을 지점을 알려 주는 신호다.

## 오늘 하루의 검증 — 규칙이 숫자로 버티는지 확인

과제 제출 이후, 이 규칙을 여러 방법으로 실제 스트레스 테스트했다.

**검증 방법 3가지**
1. **30년 몬테카를로** — 통계 모델로 1,000개의 "있을 수 있는 미래"를 만들어 확률 분포 확인
2. **실제 TQQQ 역사(2010.2~2026.7, 16.4년)** — 진짜 있었던 하루하루로 백테스트
3. **블록부트스트랩** — 실제 16.4년치 데이터를 무작위로 재조합해 1,000개의 "있을 법한 30년" 생성

**핵심 발견**
- **TQQQ 비중 상한(자산의 35~40% 초과 시 자동 트림)**과 **커버드콜 F&G 35~65 추가매수** 두 규칙이 결과를 가장 크게 개선시켰다.
- 커버드콜을 "상승 제한된 안전자산"으로 가정하면 실제(계좌 데이터)와 크게 어긋난다 — 실측 확인 결과 **베타 약 0.9(나스닥100의 90% 추종) + 연 15% 배당**이 훨씬 현실에 가깝다.
- **성장률 가정이 결과를 지배한다.** 최근 15년(2011~2026) 실측 CAGR(19.7%)은 나스닥 역사상 예외적으로 좋았던 구간이라 그대로 30년에 쓰면 비현실적으로 낙관적인 숫자(MDD −83%대)가 나온다. **30년 전체 실측 평균(14.4%)**이 더 현실적인 기본 가정이다.
- **현실적 기대치:** 나스닥 CAGR 14~15% 가정, 중앙값 기준 30년 뒤 원금의 10~25배, **MDD −45~50%대**는 반드시 감수해야 한다.

**남은 한계** — F&G 신호는 실제 CNN 지수 원본이 아니라 가격 데이터에서 역산한 대체 지표를 계속 사용했다. 방향성은 검증됐지만, 정밀한 타이밍 정확도는 원본 데이터로 재검증이 필요하다.

## 넉 달 학습으로의 확장 (개인 문제 로드맵)

이 신호 판단 자동화가 출발점이고, 이 서비스 전체는 여기서 가지를 뻗는다.

1. **신호 생성 자동화** (이번 과제) — 매일 아침 매매 신호 리포트
2. **고객 Q&A 에이전트** — 신호의 근거를 고객에게 설명
3. **ETF 분배·세금 임계 계산** — 커버드콜 배당의 과세표준·금융소득종합과세 한도 관리
4. **완전 자동 실행** (M11 해커톤) — 증권사 API로 신호 → 실제 주문까지

컴퓨팅 사고법이 못 푸는 자리는 남겨 둔다. "지금이 정말 극도의 공포인가", "이 규칙을 예외적으로 멈춰야 하는 국면인가" 같은 **가치·맥락 판단**은 20년 경험을 가진 사람의 몫이다. AI가 잘게 쪼갤 수 있는 일은 AI에게 맡기고, 사람은 이 깊은 판단으로 옮겨 간다.
